### E46 -- Fleet observability: run registry, per-run sinks, `kazi dashboard` starmap (P1, ADR-0057)

The operator runs several concurrent sessions, each driving its own
`kazi apply`, and the fleet is a black box: the LiveView surface is a
dev-server view of ONE BEAM node, a one-shot CLI run has no visible surface at
all, the inner-harness transcript is discarded (the read-model keeps the
predicate vector and ADR-0046 counters, not what the agent DID between
iterations), and a dead run leaves nothing that distinguishes "converged and
exited" from "crashed mid-iteration". ADR-0057 closes this with three pieces
that reuse what exists rather than rebuild it: a **run registry** with
heartbeats in the shared read-model (every instance on the machine already
writes one SQLite file); **per-run append-only JSONL sinks** (`events.jsonl` +
a redacted `transcript.jsonl` tee of the harness stream) so peeking equals
tailing -- live, post-mortem, and across crashes; and a **`kazi dashboard`**
verb that boots the web endpoint in a standalone read-only fleet mode. Home
view is the operator-chosen **starmap** (the goal DAG in topological wave
bands via the `--explain` frontier computation, node states landed /
converging / claimed / pending / stuck per ADR-0055, run tags, and a ranked
attention queue fed by the existing stuck/flake/regression detectors);
per-goal drill-in is the **convergence heatmap** (predicates x iterations),
an iteration scrubber, and **transcript peek** with tool-call folding.
Read-only projection throughout (ADR-0011); no control plane in this epic.
Subsumes T20.8 (marked superseded in plans/E20.md). NATS fan-in stays in
Slice 3; this dashboard becomes its first real consumer when that slice
arrives.

Existing seams reused: the shared read-model (`goal_summary` + `iterations`
with full predicate-vector history + ADR-0046 counters), the KaziWeb LiveView
assets (`dag_live`, `history_live`, `lease_map_live`), the loop detectors
(`stuck_detector`, `flake`, `regression_detector`), the redaction layer, the
group scheduler's frontier/`--explain` machinery (ADR-0027/0028), and the
E44 `landed` state (ADR-0055) as a node state.

**Waves.** Wave A (T46.1-T46.3) is the data spine and has no UI. Wave B
(T46.4-T46.8) is the surface and pipelines off Wave A per task deps -- T46.4
can start once T46.1 lands. Wave C (T46.9-T46.10) is docs + the live proof.
