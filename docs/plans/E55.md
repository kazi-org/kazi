# E55 -- Teamwork is first-class: installed delivery, a bounded digest, and a board (P1, ADR-0071/0072/0073)

Back-link: [docs/plan.md](../plan.md) (master index). Sibling epics: [plans/](.)

fidelity: executable

E51 shipped the bus and it works: a `kazi daemon` carrying a dozen live
sessions across two machines and three projects, with cross-machine presence,
facts, and directed messages all round-tripping on the released binary. Adoption
still failed, and a 2026-07-16 observation pass found three structural causes --
none of which is agent discipline.

**Delivery was never built.** ADR-0067 point 6 shipped the turn-boundary hook as
a documented recipe. Its observed install rate is zero: no hook on either
machine, with the daemon up 16h. So the operator is the delivery mechanism, and
the reported symptom ("I am having to remind the agents over and over") is the
design working as specified. Two enhancement passes on the skill's bus section
changed no behaviour, because a recipe cannot fire itself.

**The digest protects the wrong reader.** `Kazi.Bus.Digest` is reached only from
the TTY branch of `emit/3` (`cli.ex:3951/3974/4010`), so `--json` and every
`kazi_bus_*` MCP tool -- the only paths agents and the documented hook use --
return the full transcript. A single `bus peek --json` on the live bus returns
the whole backlog verbatim, including two ~10-20 KB filler messages. E51's
R-E51-4 named this exactly; all three of its mitigations were raised (the 1 KiB
cap, lifted to 64 KiB for a real document-carrying need), deferred (T51.4), or
blocked on the deferred one (T51.6).

**Teams need state, not a stream.** Two project teams independently abandoned
the bus for hand-rolled append-only markdown blackboards ("coordination has
MOVED to the file blackboard ... re-read at every turn boundary") and did not
return after the blocking bug was fixed. A stream answers "what changed since I
looked"; a team needs "what is true now". `fact`'s last-value-per-subject
retention has been a board since day one with no verb to render it -- while the
fallback files are machine-local, so leaving the bus cost those teams the one
property it uniquely has. The duplicate work that follows is a lock-VISIBILITY
failure: claims live in `refs/claims/*` that the bus never hears about, so a
lead hand-maintains ownership in prose and a session declared closed picks up
its replacement's stream.

ADR-0071 (delivery is installed), ADR-0072 (the digest bounds context at render,
keeping the document-carrying limits), and ADR-0073 (the board projects roster +
ownership + current facts) decide the fixes. This epic builds them.

Acceptance: on the RELEASED binary, across two machines, a session started in a
project with hooks installed learns the team's state without a human telling it
to -- it registers, sees the board (roster, claims, current facts) at session
start, and receives bounded digests at turn boundaries, costing nothing when the
bus is quiet. `bus read --json` and every MCP bus tool return a digest bounded to
N lines regardless of backlog depth or message size, with `--full` escaping and
`bus get <id>` pulling a document deliberately. `kazi bus board` renders current
state cursor-free. A claim taken via `/claim` is visible on the board, and a
claim still succeeds identically with the daemon down. Sessions are addressable
by name. The dashboard shows the live roster instead of "No instances present".
Token cost is measured and recorded honestly in `docs/devlog.md`.

Supersedes within E51: **T51.4** (server-side digest deepening) is absorbed --
its brief predates the discovery that `--json` bypasses the digest entirely, so
as written it would deepen a digest that never reaches an agent; it lands as
T55.1 (the inversion + the render bound) plus T55.7 (the daemon move, its
original intent). **T51.6** (two-session live dogfood) is absorbed by T55.10, a
superset. **T51.5** stays in E51: it keeps run-lifecycle mirroring, and its
presence-rail half consumes T55.3's source wiring -- see R-E55-5, and do not
implement presence rendering twice.

Not in scope (owned elsewhere): `bus watch`'s backlog bug is **T54.9** (#1097) --
E55 must not touch `watch/1`. Per-task goal-files are authored by `kazi plan`
when each task starts.

## Wave C -- server-side, claims, and the payload (parallel)

- [ ] T55.7 Digest assembly moves into the daemon (ADR-0072 d5; absorbs T51.4's original intent): the client sends `read` over the control socket; the daemon pulls the consumer, aggregates last-value facts per topic, collapses repeated presence/intent, and enforces verbatim-only-for-directed-or-interrupt before the bytes ever reach a client. This is ADR-0067 point 5's own load-bearing argument -- "only a server can aggregate before the tokens are spent" -- and it is what keeps the CLI, the MCP tools, and the T55.9 hook from re-implementing the bound three times. Carries T51.4's `bus read --since <cursor>` debugging escape. ## Waves

- **Wave E55-A (3 parallel):** T55.1 (the bound), T55.2 (the installer), T55.3 (the dashboard). The two highest-leverage items plus the one that touches no shared file.
- **Wave E55-B (3 parallel):** T55.4 (board), T55.5 (identity), T55.6 (`bus get`). T55.5 has no deps and may start in Wave A if a session is free and R-cli-ex is uncontended.
- **Wave E55-C (3 parallel):** T55.7 (daemon digest), T55.8 (claims), T55.9 (hook payload).
- **Wave E55-D:** T55.10 closes the epic post-release.

**Ordering constraint that is not a dep edge:** do NOT enable hook delivery to
real sessions before T55.1 has landed. The installer (T55.2) and the bound
(T55.1) are independently correct and genuinely parallel -- different files
entirely -- but pushing today's raw-transcript read into every session at every
turn boundary would automate R-E51-4 rather than fix it. T55.9 carries the dep
edge; Wave A carries the discipline.

**Shared-file locks.** `lib/kazi/cli.ex` (R-cli-ex) and `docs/session-bus.md`
(R-session-bus-doc) are touched by several tasks. Each touch is small and
additive -- one `@bus_verbs` entry, one `execute_bus/3` clause, one help line,
one flag doc -- so take the lock for the wiring commit only and release it
immediately; do not hold it across implementation. T55.3 and Waves C/D need no
locks at all.
