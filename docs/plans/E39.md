### E39 -- Orchestrator-driving ergonomics: close the plan -> approve -> apply loop over `--json` (P1, ADR-0049)

The T15.9 nested-loop dogfood (orchestrator -> kazi -> a local model via opencode)
drove the full `plan -> approve -> apply` spine over `--json` and proved the inner
loop works, but surfaced four concrete points where kazi is awkward to drive as a
tool. This epic resolves them so an outer orchestrator can drive the spine end to
end with zero prose-scraping and zero goal-file reconstruction. Source friction is
recorded in `docs/devlog.md` (2026-06-28, T15.9); the decision is ADR-0049.
