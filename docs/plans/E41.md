### E41 -- Crystallize discovered truth: an open use-case-manifest schema, harness-driven discovery, and prod-log predicate correlation (P2, ADR-0051)

`I \ A` / `A \ I` (ADR-0021) has three declared `I`-sources (OpenAPI/gherkin,
prose, behavior specs) but no empirical one: nothing turns "we explored the
live system and found this works / this is broken" into a standing predicate.
An earlier draft of this epic assumed the operator's personal `/verify`/
`/qualify` Claude Code skills as the source -- corrected: those are not
installed by `brew install kazi` and most kazi users will never have them. This
epic instead (a) defines an OPEN schema any tool can emit, (b) gives kazi its
OWN harness-driven discovery step so the capability works for every user
regardless of installed skills (reusing the exact seam `Kazi.Adopt.enrich/2`
already proved out, ADR-0013 §4), and (c) crystallizes either source into
predicates via a new importer, plus an unrelated, independent prod-log
correlation provider extension.

Acceptance: `kazi spec discover` drives the user's configured harness to
produce a documented use-case-manifest + wiring-gap report with NO external
skill required; `kazi spec import --from-usecase-manifest` upserts either that
or any externally-produced manifest of the same schema into a goal -- proven
use cases become held-true predicates, wiring gaps feed the existing
surface-coverage meta-predicate (T13.5); a predicate can opt into prod-log
correlation so a false-green result gets flagged.
