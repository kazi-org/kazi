# E50 -- Safe concurrent work: serial worktree indirection, wave checkpoints, fleets over goal-files (P1, ADR-0065)

Back-link: [docs/plan.md](../plan.md) (master index). Sibling epics: [plans/](.)

fidelity: executable

Issue #937 (filed after a real data-loss incident) left two structural gaps
open once its tripwires shipped (#940 primary-worktree refusal, #942
duplicate-run refusal, #936's minimal `frontier_complete` stream event): a
serial `kazi apply` still edits `--workspace` in place -- the operator
hand-builds a worktree per goal, every time, by convention (five times in the
2026-07-09 batch on this repo alone) -- and nothing coordinates SEVERAL
goal-files as one dependency-aware unit (sequencing, conflict-ordering, and
concurrency throttling for that batch were all manual). ADR-0065 decides:
serial apply becomes the 1-partition degenerate case of the E21 parallel
scheduler (kazi creates and owns the task worktree; `--workspace` means "the
base to integrate onto"; `--in-place` opts out), integration follows T21.5
(never a tree-wide landing), and `--fleet` runs a DAG of goal-files reusing
E21's worktrees/leases/blast-radius/budget machinery one level up, with the
#936 supervised-checkpoint mode (`--pause-between-waves`, resumable) landing
as one mechanism covering both wave and fleet-node boundaries. Single-node,
NATS-free (registry-substrate coordination only).

Acceptance: a serial executing apply against a base ref never mutates the
caller's checkout (worktree-isolated by default, `--in-place` to opt out); a
paused parallel/fleet run stops at a declared boundary and resumes; a
directory of goal-files with `depends_on` edges executes as a pipelined DAG
with overlap-serialized concurrency; a stale base is loudly visible
(`--base <ref>` explicit, behind-upstream warned, never an implicit network
fetch); every claim ExUnit-covered plus one live dogfood re-running the
2026-07-09 batch shape as a fleet.

## Risks

| id | risk | p | impact | mitigation |
|---|---|---|---|---|
| R-E50-1 | Worktree-by-default breaks callers whose predicates assume the workspace path IS the edit site (relative-path scripts, absolute-path fixtures). | Med | Med | `--in-place` opt-out + a deprecation window; predicates already run inside the worktree the loop dispatches to, so only path-hardcoding callers notice; release notes call it out. |
| R-E50-2 | Scope-overlap inference over-serializes fleets with broad `[scope]` globs, silently killing parallelism. | Med | Low | `--explain` renders every inferred edge with its overlapping paths; explicit `depends_on = []` disables inference per pair; docs teach narrow scopes. |
| R-E50-3 | Resume tokens add persistent scheduler state that can go stale (binary upgrade mid-pause). | Low | Med | Token embeds schema_version + goal-set hash; a mismatched resume refuses loudly and says to re-run; state lives in the existing read-model, reaped by existing staleness machinery. |
| R-E50-4 | The behind-upstream warning nags callers who deliberately build on a pinned/old base, training them to ignore it. | Low | Low | `--base` states intent and silences the warning; it is a warning, never a refusal; the message names the exact SHAs so a surprised operator can verify in one command. |
