# E63 -- Dashboard UX: truthful attention, balanced widgets, legible goal views, operator questions (P1)

Back-link: [docs/plan.md](../plan.md) (master index). Sibling epics: [plans/](.)

fidelity: executable

Operator feedback (David, 2026-07-17, live on the v1.212.0 dashboard at
http://localhost:4050 immediately after the PR #1366 mission-control changes
shipped). Four concerns, verbatim intent:

1. **NEEDS ATTENTION shows stale/old runs** -- the section's purpose is unclear
   and its contents are untrustworthy. Root cause is known territory: the
   section is `Kazi.Attention.Queue.build/2` (`lib/kazi/attention/queue.ex:105`)
   ranked over registry runs, and the registry carries ghost `running` rows
   with days-stale heartbeats (#1155, E60/T60.2). An attention queue that
   ranks phantoms trains the operator to ignore it -- the worst outcome for
   an attention surface.
2. **The new widgets (PR #1366: project badge, age/last-active row,
   state/repo/time chip filters) feel unbalanced** against the original
   mission-control visual system. Wants design explorations to pick from,
   not a single take.
3. **The goal drill-in and history views are not user friendly** (e.g.
   `/goals/<goal>/drillin` and `/goals/<goal>/history`) -- it is unclear what
   they are supposed to show; history is harder still.
4. **The dashboard should answer the operator's four standing questions**:
   what is my fleet working on; where can I help; what is blocking them; how
   much longer until projects complete (explicitly acknowledged as the tricky
   one). Overlaps E60's portfolio/attention work (T60.3/T60.4) -- this epic
   must compose with it, not duplicate it.

Design-direction selection is HUMAN-GATED: David picks from presented
explorations before any implementation task builds the chosen direction.

Exit criteria: NEEDS ATTENTION contains only actionable, heartbeat-fresh
entries and states what qualifies an entry; David has selected a widget
design direction from real explorations and it is implemented; the drill-in
and history views each lead with a one-line answer to "what am I looking
at"; the dashboard answers the four operator questions with honest data
(no fabricated ETAs -- ADR-0046 honest-unknown applies to time estimates).
