# E51 -- Session coordination bus: `kazi daemon` + bus primitives on the JetStream substrate (P1, ADR-0067)

Back-link: [docs/plan.md](../plan.md) (master index). Sibling epics: [plans/](.)

fidelity: executable

Operators run several concurrent driving sessions per project and several
projects per machine; the E50 fleet direction multiplies the kazi processes
each session supervises. Sessions cannot talk: no presence ("is the session
converging goal X abandoned?"), no intent broadcast (the #1018/#1019
release-window incidents had no warning channel), no shared facts, no directed
messages. ADR-0067 (Accepted 2026-07-10) decides: a new `kazi daemon`
supervises a local `nats-server` and hosts a session bus on the ADR-0004
JetStream substrate -- presence as TTL'd KV, facts as last-value subjects,
intents/events as an age-bounded stream, directed messages via per-session
durable consumers. Token economy is enforced server-side (digests, ~1 KB
message cap, severity gating). Convergence NEVER depends on the bus; every
surface no-ops cleanly without a daemon. The repo already carries the
plumbing: `{:gnat, "~> 1.9"}` is a dep, `Kazi.Coordination.Lease.Nats` +
`Kazi.Coordination.Transport` exist, and test_helper.exs already gates
`:nats`-tagged tests behind `NATS_URL`.

Acceptance: `kazi daemon start|status|stop` lifecycle works over a Unix
socket with a version handshake; with the daemon up, `kazi bus post/read/
who/tell` round-trip through JetStream with per-session durable-consumer
cursors and presence TTL; without a daemon every bus surface reports cleanly
and exits nonzero-gracefully; a goal converges identically with the daemon
down; docs + mcp tools land with the code.

- [x] T51.1 `kazi daemon` skeleton -- lifecycle + Unix-socket control plane: new command group `kazi daemon start|stop|status` (wire like `kazi dashboard`, cli.ex ~1013). `start` runs the daemon supervision tree in the foreground (operator backgrounds it; same convention as the dashboard), writes `~/.kazi/daemon/daemon.pid`, and listens on a Unix domain socket `~/.kazi/daemon/daemon.sock` (`:gen_tcp` with `{:local, path}`); a line-JSON control protocol answers `ping` with `{ok, vsn, uptime_s}`. `status` connects, prints the handshake (and `--json` emits it raw); `stop` sends a clean shutdown; both exit with a clear one-line "no daemon running (socket absent/refused)" when down. Stale socket/pidfile from a dead daemon is detected (connect refused -> remove + report), never a false "running". NO NATS yet -- this task is pure lifecycle. - [x] T51.2 Bus MVP over a supervised nats-server: daemon supervises `nats-server` (resolved from PATH or `--nats-bin`; `--nats-port`, default 4223 to avoid colliding with an operator's own NATS) as a Port with JetStream enabled (`-js`), restart-on-crash, health-checked via Gnat connect at boot. Provision on boot: stream `KAZI_BUS` (subjects `bus.>`, max_age 24h, max_msg_size 1024) and KV bucket `kazi_sessions` (TTL 10m). New `Kazi.Bus` client module + CLI verbs: `kazi bus post <kind> <text> [--topic <t>] [--sev info|interrupt] [--scope machine|project]` publishes to `bus.<scope>.<kind>.<topic>` with provenance headers (session id via the existing session-name resolution chain, machine, ts); `kazi bus who` lists the presence KV; `kazi bus tell <session> <text>` publishes to `bus.<scope>.msg.<session>`; `kazi bus read [--json]` pulls this session's durable consumer (created on first read; name = sanitized session id) and prints a DIGEST -- counts per kind/topic for bulk traffic, verbatim lines only for directed messages and `sev: interrupt` (deeper digest logic is T51.4; the read path structure lands here). Every bus CLI invocation upserts the caller's presence key. All verbs no-op with the clear "no daemon" error when the socket is down (never touch NATS directly from the CLI -- daemon discovery via T51.1's socket handshake, which reports the NATS port). ## Waves

- **Wave E51-A (the MVP chain, serial):** T51.1 -> T51.2 -> T51.3. Runs as the T50.7 fleet dogfood (a real 3-goal DAG).
- **Wave E51-B (depth):** T51.5 after Wave A ships. T51.4 and T51.6 are SUPERSEDED by E55 (ADR-0076/0072/0073) and leave the live WBS; E55's T55.10 closes the bus story on a cross-machine dogfood.
